Site admin & monitoring

This page is public (GitHub Pages has no login) but hidden from search engines (noindex) and carries no secrets. It links to your private analytics dashboard and helps you tag ad-campaign links.

Traffic analytics dashboard

Visitor counts, countries/regions, referrers, real-time activity and ad-campaign performance. Hosted on your own subdomain (Umami). Login is required — only you see the data.

Open analytics dashboard ↗

Not set up yet? Follow docs/analytics-setup.md in the repo, then paste the Website ID into tools/build_seo.py and run it.

Ad-campaign link builder (UTM)

Paste the destination page, describe the campaign, and copy the tagged link to use in your ad. Umami reports these under Campaigns.

Quick reference

UTM conventions — keep them lowercase and consistent so reports stay clean: utm_source = where (facebook), utm_medium = type (social / cpc / email), utm_campaign = the campaign name.

Privacy / GDPR — Umami is cookieless and does not store personal data or raw IPs, so no consent banner is required. If you later switch to a tool that logs individual IPs (e.g. Matomo full-IP), add a consent notice.

Turning analytics on — deploy Umami, create a website for pacificaromania.space, copy its Website ID into UMAMI_WEBSITE_ID in tools/build_seo.py, run python3 tools/build_seo.py, then commit & push.
